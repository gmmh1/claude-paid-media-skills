# Creative Brief Generator

> **Currency & scope note (last reviewed 2026-07-19):** Platform mechanics referenced here (character limits, campaign-type names, feature availability, thresholds, benchmark figures, policy specifics) reflect general practice as of the review date above and are **not guaranteed current** — ad platforms change quickly. ## Purpose
Turn a business goal into a structured creative brief — the inputs every platform-specific creative skill (and any human creator/designer) needs before producing anything — so creative work starts from clear direction rather than vague instructions.

## Trigger Conditions
- "Build a creative brief for [campaign/product]"
- "What do I need to define before writing ads"
- "Brief a creator/designer for this campaign"

## Required Inputs
- Product/service and the specific offer for this campaign
- Target audience and awareness stage
- Campaign objective and funnel stage
- Platform(s) this will run on (affects format requirements downstream)
- Brand voice/guidelines, if established
- Deadline and available budget for production

## Core Capabilities

### Brief Structure
- **Audience**: who specifically, including awareness stage (feeds `ad-copywriting-frameworks` framework selection)
- **Offer**: what's actually being offered, with enough specificity that "special promotion" becomes "20% off first order, code WELCOME20, expires [date]"
- **Objective**: the single primary goal this creative needs to achieve — resist listing multiple competing objectives for one asset, which produces unfocused creative
- **Messaging**: the core message/angle, informed by `ad-copywriting-frameworks` and audience awareness stage
- **Deliverables**: exact format list needed per platform (e.g., "3 Meta Reels variants 9:16, 2 Google RSA asset sets, 1 TikTok UGC script") rather than a vague "some ads"
- **Timeline and budget**: production deadline and available spend, since this affects whether in-house production, stock/AI generation, or creator partnerships are realistic

### Platform-Aware Deliverable Specification
- Translating one campaign goal into the specific asset list each target platform actually needs — a single "make some ads" brief produces mismatched output; specifying "Google needs 15 headlines + 4 descriptions, Meta needs primary text + 3 video variants, TikTok needs 5 native script concepts" sets creators/AI tools up to succeed
- Flagging when the same core message needs meaningfully different execution per platform (professional register for LinkedIn vs. native/casual for TikTok) rather than assuming one asset can be reused everywhere unchanged

### Avoiding Common Brief Failures
- Vague objectives ("build brand awareness and drive sales and get followers") diluting focus — push for a single primary objective per asset/campaign
- Missing specificity on the offer, audience, or proof points, forcing the creative executor to guess
- No format/deliverable list, leaving creators to guess what's actually needed
- Brand guidelines referenced but not actually supplied, producing off-brand output that then needs rework

## Workflow
1. Gather product/offer, audience/awareness stage, single primary objective, target platform(s), and brand guidelines.
2. Draft the messaging direction, referencing `ad-copywriting-frameworks` for the appropriate persuasion structure given awareness stage.
3. Specify the exact deliverable list per platform, translating one goal into platform-appropriate asset requirements.
4. Confirm timeline and budget, flagging if production capacity doesn't match the deliverable list's ambition.
5. Hand off to the relevant platform-specific creative skill(s) for execution, or to a human creator/designer using this brief directly.

## Outputs
- Structured brief: audience, offer, objective, messaging direction, platform-specific deliverable list, timeline/budget
- Framework recommendation reference
- Flags for any specificity gaps that need resolving before production starts

## Rules
- Never let a brief carry more than one primary objective per asset — competing goals produce unfocused creative.
- Always specify exact deliverables per platform rather than a vague asset count — this is the most common cause of creative rework.
- Flag when brand guidelines are referenced but not actually supplied; don't let production start on an assumption.

- **Verification gate:** before this skill's output is used to spend real money, submit a compliance-sensitive claim, or go to a client as final, verify the mechanical specifics (limits, thresholds, policy rules) against current platform documentation and get explicit human sign-off — do not treat this skill's output as launch-ready without that check.
