## Purpose
Write the actual scripts, hook lines, captions, and creator briefs that execute a TikTok creative angle — native, conversational, and structured for a platform where the opening line decides whether anyone keeps watching.

## Trigger Conditions
- "Write a TikTok ad script for [product]"
- "Write a UGC script for a creator"
- "Write captions/on-screen text for my TikTok ad"
- "Brief a creator for a TikTok Ad"

## Required Inputs
- Product/service, offer, and the creative angle already decided (route to `tiktok-ads-creative-strategy` if not yet defined)
- Whether this is scripted for in-house talent, briefed for a creator, or written as caption/on-screen text for existing footage
- Target length (9-15s hook-driven vs. 15-34s demonstration/story)

## Core Capabilities

### Hook Lines (First 1-2 Seconds)
- The single highest-leverage line in the entire script — must create an immediate pattern interrupt, direct address, or bold claim/question relevant to the specific viewer
- Native phrasing: "POV:", direct-to-camera address, "wait for it," or a blunt problem statement — conversational TikTok vernacular outperforms polished ad copy openers
- Avoid brand name or slow setup in the opening line; get to the hook immediately

### Script Structure
- Hook → problem/context → demonstration or proof → CTA, compressed into the target length without padding
- Natural, spoken-register language throughout — scripts that read like a friend explaining something outperform scripts that read like ad copy read aloud
- Multiple short beats/cuts implied in the script structure (matching the fast-paced editing `tiktok-ads-creative-strategy` calls for) rather than one long unbroken monologue

### Captions & On-Screen Text
- Concise on-screen text reinforcing the spoken hook and key points for sound-off viewers
- Caption (the text below the video, distinct from on-screen text) as a secondary hook/context opportunity, often reinforcing urgency or a key detail not stated on-screen

### Creator Briefing
- Briefing with a hook line, key talking points, and desired CTA rather than a rigid word-for-word script — natural delivery from a creator's authentic voice consistently outperforms visibly-read copy
- Multiple take/angle requests from one briefing to generate several test variants efficiently
- Clarifying required disclosures (e.g., paid partnership labeling) as part of the brief, not an afterthought

### CTA & Offer Communication
- Direct, simple CTA language matching the platform's fast pace — avoid multi-step or complex instructions that don't fit the format's brevity
- TikTok Shop-specific CTA conventions when the campaign links to Shop (route to `tiktok-shop-organic` context for Shop-specific mechanics)

## Workflow
1. Confirm the creative angle, format length, and whether this is a full script, a creator brief, or caption/on-screen text for existing footage.
2. Write the hook line first, in native conversational register.
3. Build out the remaining structure (context → proof → CTA) within the target length.
4. Write on-screen text and caption separately, each serving a distinct reinforcing role.
5. If briefing a creator, provide hook + key points + required disclosures, not a rigid script.
6. Produce multiple hook variants for testing.

## Outputs
- Script or creator brief, hook-first
- On-screen text plan
- Caption copy
- Multiple hook variants for testing

## Rules
- Never open with brand name, slow setup, or polished ad-copy phrasing — the hook must read as native platform content immediately.
- Write in spoken, conversational register throughout; avoid formal ad-copy phrasing at any point in the script.
- Include required disclosure language (paid partnership, etc.) in creator briefs as a standard, not optional, element.

- **Verification gate:** before this skill's output is used to spend real money, submit a compliance-sensitive claim, or go to a client as final, verify the mechanical specifics (limits, thresholds, policy rules) against current platform documentation and get explicit human sign-off — do not treat this skill's output as launch-ready without that check.
